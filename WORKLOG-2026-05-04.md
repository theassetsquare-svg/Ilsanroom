# 2026-05-04 작업 로그 — "한번에 도달" 캠페인

> **목표**: 사용자가 어떤 URL로 진입하든 다른 페이지를 거치지 않고 한 번에 도달하게 만든다. 모바일·PC 동등한 경험.

## 사용자 클레임

> "사이트에 들어갈 때 왜 한번에 가지 않고 다른 페이지에 갔다가 가는 거야?"
> "모든 페이지에 갈 때 한번에 갈 수 있게 강력하게 설정해."

## 진단 — "거치는 페이지" 깜빡임의 두 가지 원인

### 원인 1 — 짧은 URL 2-bounce (해결: commit `8a110b2`)
- 기존 `public/_redirects`는 SPA fallback 한 줄(`/*  /index.html  200`)뿐
- `/ilsanroom`, `/rooms/ilsanroom` 등 짧은 URL → SPA fallback으로 홈 HTML 수신 → React Router가 클라이언트에서 정식 URL로 다시 라우팅
- 사용자 인식: "홈이 잠깐 보였다가 정식 페이지로 이동" = 2-bounce

### 원인 2 — instant-hero 풀스크린 오버레이 (해결: commit `81d258c`)
- `index.html`의 `<div id="instant-hero">`가 `position:fixed; inset:0; z-index:0`로 풀스크린을 덮음
- 어두운 그라디언트 + H1 + "실시간 정보 로딩" 스피너 표시
- React 350ms 후 부팅 → `MainLayout` `useEffect`가 페이드아웃 → 진짜 페이지
- 사용자 인식: "로딩 페이지 → 실제 페이지" 2단계
- prerender SSR 본문은 `<div class="ssr-seo">`로 시각 숨김 처리 (clip:rect(0,0,0,0)) → 사용자 눈엔 안 보임

## 조치

### A. `_redirects` 자동생성 (commit `8a110b2`)

`scripts/prerender-seo.mjs` 5b 섹션에서 빌드 시 venue별 301 매핑 + 정적 파일 200 + SPA fallback을 자동 작성.

```
# Before  (2 lines)
/api/* /api/:splat 200
/*     /index.html 200

# After   (330 lines)
/llms.txt        /llms.txt        200
/llms-full.txt   /llms-full.txt   200
/sitemap.xml     /sitemap.xml     200
/robots.txt      /robots.txt      200

# venue 짧은 URL → 정식 URL 301 (318줄)
/rooms/ilsanroom  /rooms/ilsan/ilsanroom/  301
/rooms/ilsanroom/ /rooms/ilsan/ilsanroom/  301
/ilsanroom        /rooms/ilsan/ilsanroom/  301
/ilsanroom/       /rooms/ilsan/ilsanroom/  301
... (120 venue × 2~4 매핑)

/api/*  /api/:splat   200
/*      /index.html   200
```

**효과**: 엣지(Cloudflare) 단계에서 1-bounce 301 → 정식 URL → 200. 클라이언트 라우팅 우회 + SEO PageRank 정식 URL로 합산.

### B. instant-hero 영구 제거 (commit `81d258c`)

3개 파일에서 `instant-hero` 흔적 삭제 (-47 lines):

| 파일 | 변경 |
|---|---|
| `index.html` | `<div id="instant-hero">…</div>` 통째 삭제. `<div id="root">`만 남김 |
| `scripts/prerender-seo.mjs` | 페이지별 instant-hero 동적 주입 코드(187~201줄) 제거 |
| `src/layouts/MainLayout.tsx` | `useEffect`에서 instant-hero 페이드아웃·제거 로직 삭제 |

**효과**: 첫 페인트 = body 라이트 배경(`bg-[#F5F5F5]`) 짧게 → React 부팅 → 진짜 페이지. 사용자 인식: "자연스러운 로드" 1단계. SSR 본문(`ssr-seo` 시각 숨김)은 그대로 → 검색엔진 SEO 손실 0.
