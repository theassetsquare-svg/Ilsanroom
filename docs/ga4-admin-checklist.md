## ✅ 1. 데이터 보관 14개월
- 관리 → **데이터 수집 및 수정** → **데이터 보관(Data retention)**
- "사용자 및 이벤트 데이터 보관"을 **14개월** 선택 → **저장**
- (기본은 2개월 — 14개월로 올려야 오래된 추세 비교 가능)

## ✅ 2. 시간대 = Asia/Seoul
- 관리 → **속성 설정(속성 세부정보)**
- "보고 시간대" → **(GMT+09:00) 대한민국 시간** 선택 → **저장**
- 통화는 **대한민국 원(KRW)** 권장(매출 페이지 없으니 표시용)

## ✅ 3. 내부 IP(사무실·집·대표님) 제외
- 관리 → **데이터 수집 및 수정** → **데이터 스트림** → 웹 스트림 클릭
- → **태그 설정 구성** → **내부 트래픽 정의** → **만들기**
  - 규칙 이름: `internal` / IP 주소 = 사무실·집 공인 IP 입력(검색창에 "내 IP" 치면 나옴)
- 그다음 관리 → **데이터 수정(데이터 필터)** → **Internal Traffic** 필터를 **테스트 → 사용(활성)**으로 변경
- (코드가 이미 관리자 계정·자동화·내부 호스트 트래픽은 GA로 안 보냅니다. 이 IP 필터는 비로그인 상태로 직접 볼 때까지 잡는 2차 방어)

## ✅ 4. 봇·스파이더 필터
- GA4는 **알려진 봇/스파이더를 기본으로 자동 제외**합니다(켜고 끄는 토글 없음 = 항상 ON). 추가 작업 없음, 확인만.
- 코드 쪽에서도 `googlebot/yeti/gptbot/headlesschrome/puppeteer/lighthouse` 등과 `navigator.webdriver` 자동화를 GA 발송에서 막아둠.

## ✅ 5. 핵심 이벤트(참여 중심 — 판매전환 아님)
- 놀쿨은 "구매"가 아니라 **참여**가 목표입니다. 아래 이벤트를 **핵심 이벤트로 표시**:
- 관리 → **데이터 표시 → 이벤트** → 각 이벤트 우측 **"핵심 이벤트로 표시"** 토글 ON
  - `sign_up`(가입) · `login`(로그인) · `post_create`(글쓰기) · `share`(공유)
  - `invite_open`(초대) · `search`(검색) · `search_no_result`(검색했지만 없음=만들 페이지 신호) · `scroll_100`(끝까지 읽기)
- ⚠️ 이벤트는 **실제로 한 번이라도 발생한 뒤에** 목록에 나타납니다. 안 보이면 사이트에서 그 행동이 일어날 때까지 대기(24~48h).

## ✅ 6. "스크롤도달 0%" 해결 — scroll_100 등록
- 옵티마이저 메일에 모든 페이지 "스크롤도달 0%"로 뜨는 이유 = `scroll_100`이 아직 **핵심 이벤트/맞춤 측정기준으로 집계 설정 안 됨**.
- 위 5번에서 `scroll_100`을 **핵심 이벤트로 표시**하면 다음날부터 집계 시작(전파 24~48h).
- (검색어는 이미 등록됨: 관리 → **맞춤 정의** → 맞춤 측정기준에 `search_term`(이벤트 범위) 존재 확인)

## ✅ 7. 향상된 측정 위생
- 관리 → **데이터 스트림** → 웹 → **향상된 측정**
- "페이지 조회"는 **우리가 수동 발송**(코드 `send_page_view:false`)하므로 중복 안 나게 유지 — 스크롤/사이트 검색/이탈 클릭 등 나머지는 기본 ON 유지.

## 🔒 8. PII 2차 방어 — 데이터 수정(Data Redaction)
> 코드에서 이미 이메일·전화·URL 쿼리를 GA로 안 보냅니다(scrubPii 단일 관문 + 빌드 게이트).
> GA 단에서 한 번 더 = **벨트+멜빵**. 켜두면 안전합니다.
- 관리 → **데이터 스트림** → 웹 → **태그 설정 구성** → **더보기** → **데이터 수정(Redact data)**
- **이메일 주소 수정** 켜기
- **URL 쿼리 매개변수 수정** 켜기 → 매개변수 목록에 입력:
  `q, email, phone, tel, name, token, user_email, password`
- **저장**

## 🔒 9. 서비스계정은 "뷰어"(읽기전용) 그대로 — 절대 승격 금지
- 관리 → **속성 액세스 관리**
- `gsc-mcp@theasset-gsc.iam.gserviceaccount.com` 역할이 **뷰어(Viewer)** 인지 확인
- ⚠️ **편집자/관리자로 올리지 마세요.** 놀쿨 자동 감시는 **읽기만** 필요합니다(쓰기 권한 0 = 사이트/데이터 조작 불가 = 안전).
- 자동화 토큰 스코프도 `analytics.readonly`(읽기 전용)로 고정돼 있습니다.
